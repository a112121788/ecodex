# ECode 1.0.0 发布说明

ECode 1.0.0 是第一个 Windows 原生 SuperTerminal 稳定版。它把终端 Workspace、浏览器 Surface、脚本化自动化、会话恢复和 Windows 安装更新整合到一个桌面工作流中。

## 推荐下载

| 产物 | 适用场景 | 说明 |
|---|---|---|
| `ecode-win-x64-sc` | 大多数 Windows 10/11 用户 | self-contained app 目录，不需要共享 .NET runtime。 |
| `ecode-cli-win-x64` | 自动化脚本和 CI | 仅命令行包，提供 `ecode` 命令。 |
| Velopack 安装器/feed | 需要自动更新的用户 | 支持 `ecode update check` 与 `ecode update install`。 |
| Inno Setup 安装器 | 传统桌面安装 | 创建快捷方式，卸载保留 `%USERPROFILE%\.ecode`。 |
| MSIX 包 | 企业分发 | 适合受管环境。 |

详细步骤见 [安装](../installation.md)。

## 新特性

- **Terminal Workspace**：支持多个 Workspace、Surface 与 split Pane。
- **会话持久化**：保存布局、terminal snapshots、browser metadata 与 trusted resume bindings。
- **浏览器 Surface**：在终端旁打开 WebView2 浏览器，保存 URL 与 history。
- **浏览器 API**：通过 `ecode browser` 与 `ecode.v2` 自动化 snapshot、locator、click、fill、hover、press、eval、screenshots、cookies、storage、console、dialog、download、highlight。
- **短引用与 v2 命令行**：使用 `window:N`、`workspace:N`、`surface:N`、`pane:N` 定位对象。
- **Windows 集成**：PATH、PowerShell completion、Windows Terminal profiles、doctor、update 与发布产物。

## 首次运行

安装后运行：

```powershell
ecode doctor
ecode setup status
ecode completion powershell
```

创建 Workspace 并尝试浏览器 Surface：

```powershell
ecode workspace create --name demo
ecode pane split right
ecode browser new https://example.com
ecode status --json
```

## 升级注意事项

- 替换文件前先停止 `ecode-app.exe`、`ecode-daemon.exe` 和旧终端会话。
- 如果依赖自定义恢复绑定，请备份 `%USERPROFILE%\.ecode\session.json` 与 `%USERPROFILE%\.ecode\resume.json`。
- 卸载与更新默认保留 `%USERPROFILE%\.ecode`。
- 安装后运行 `ecode doctor` 检查 ConPTY、WebView2、PATH、daemon 与 config 目录。

## 已知限制

- ConPTY 需要 Windows 10 1809 / build 17763 或更新版本。
- 浏览器 Surface 需要 WebView2 Runtime。
- `Release` 构建会发布 `ecode-perf-report`，包含冷启动、`ecode status`、`browser.snapshot`、session save 预算行。
- 浏览器自动化对非 P0 的 network、viewport、geolocation、trace、screencast 操作返回 `not_supported`，详见 [浏览器 API](../browser-api.md)。
- 发布阻塞项见 [发布就绪](../release-readiness.md)；1.0 门槛要求 P0 = 0、P1 <= 3 且有规避方案。
