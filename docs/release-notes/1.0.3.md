# ECodex 1.0.3 发布说明

ECodex 1.0.3 聚焦 Windows 桌面生命周期一致性：关闭按钮明确退出主进程，重复启动只激活已有窗口，最小化后仍保留任务栏入口并继续后台运行。

## 推荐下载

| 产物 | 适用场景 | 说明 |
|---|---|---|
| `ecodex-win-x64-sc` | 大多数 Windows 10/11 用户 | self-contained app 目录，不需要共享 .NET runtime。 |
| `ecodex-cli-win-x64` | 自动化脚本和 CI | 仅命令行包，提供 `ecodex` 命令。 |
| Inno Setup 安装器 | 传统桌面安装 | 简体中文安装 / 卸载界面，创建快捷方式，卸载保留 `%USERPROFILE%\.ecodex`。 |
| MSIX 包 | 企业分发 | 适合受管环境。 |

详细步骤见 [安装](../installation.md)。

## 窗口生命周期

- **关闭按钮会退出**：点击右上角 X 会保存当前会话并退出 `ecodex-app` 主进程；daemon 托管终端是否保留仍由 `PreserveDaemonSessionsOnClose` 决定。
- **最小化保留任务栏**：最小化后 ECodex 继续后台运行，任务栏按钮仍保留；再次点击任务栏图标会把窗口带回前台。
- **单实例保护**：`ecodex-app` 只允许一个长期运行实例；重复启动会通过 `Global\ECodexMainApp` 互斥体退出新进程，并请求已有窗口聚焦。
- **任务栏激活一致**：安装版快捷方式和主进程统一使用 AppUserModelID `ECodex.App`，固定任务栏图标、任务栏按钮和托盘入口都指向同一个已有窗口。

## 运维与维护者改进

- `scripts/smoke-toast-activation.ps1` 覆盖 AppUserModelID 与任务栏 / Toast 激活检查，便于发布前记录 Windows 手动 smoke 证据。
- 文档、spec 与 source tests 已同步关闭、最小化、单实例和后台运行语义，避免后续回退到“最小化隐藏到托盘”的旧行为。

## 升级注意事项

- 替换文件前建议停止 `ecodex-app.exe`、`ecodex-daemon.exe` 和旧终端会话，避免 Windows 文件锁。
- 如果从未绑定 AppUserModelID 的裸 exe 路径手动重复启动，系统可能短暂创建启动进程；该进程会立即退出并把已有窗口带回前台。
- 安装后运行 `ecodex doctor` 和 `ecodex setup status` 检查 PATH、daemon、WebView2、PowerShell hook 与配置目录。
- 卸载与更新默认保留 `%USERPROFILE%\.ecodex`，不会删除现有 session、resume、command log 或用户 skills。

## 发布门槛

- 发布前仍按 P0 = 0、P1 <= 3 且 P1 必须有 workaround 的门槛验收。
- 如果安装、任务栏激活、Toast、WebView2 或 ConPTY 行为异常，优先参考 [故障排查](../troubleshooting.md) 和 [发布就绪](../release-readiness.md)。
